What this page is. The reserved /buy/saved-vendors route had no contract behind it. The 2026-07-16 owner ruling folded "Saved Vendors" into ⭐ Preferred — the already-frozen operations.vendor_favorites (M4 / BC-OPS-1), a flag on the buyer↔vendor relationship. So there is no new module, no new contract, no new escalation: the ⭐ toggle here is exactly ops.set_vendor_favorite.v1 / ops.clear_vendor_favorite.v1. Visiting #/saved-vendors redirects here to the ⭐ Preferred view. This directory also absorbs the shipped Vendor CRM (P-BUY-26/27) — the record, notes, rating, status and history all live on the same relationship.

Add Vendor

Two separate journeys: save a marketplace vendor to your directory, or create a private vendor the buyer maintains — one at a time, or many by pasting a list.

Two journeys, never conflated (D4 / MINOR-1). Search Marketplace → “Save vendor” is the friendly label for the single ops.set_vendor_favorite.v1 action, which creates or reuses the buyer_supplier_relationships container. Add Private / Paste → ops.create_private_vendor.v1 creates a directory record. There is no “Save then Prefer” sequence.
Search the marketplace
Unified search (spec §5). One query composes the M2 public vendor search with ops.list_private_vendors.v1. Marketplace hit → “Save vendor” = ops.set_vendor_favorite.v1 (container auto-created, Doc-4F §F4.5). Own private hit → open the record (no duplicate).
Create private vendor
Field governance (spec §6 / D2 / D5). FROZEN fields map to real private_vendor_records columns; GATED-R5.1 extras persist in an interim details_jsonb shape. Products & services + category matching are prototype-only (D5): no M2 category ID is persisted in M4 until the R5 ruling. Progressive disclosure — the essentials first, the rest under “Add more company information”. Nothing posts here.

Paste multiple vendors

Copy rows from Excel, Google Sheets or a Word table and paste them below. Review and confirm — nothing is created until you do.

Clipboard paste is NOT bulk import (D3 · MAJOR-2). Parsing, mapping, validation and preview are client-side presentation. On confirm, each eligible row would create a record through the frozen single-create ops.create_private_vendor.v1 (source = excel) — no batch/atomicity/import-job semantics are claimed. Dedicated batch create, file upload, Word/PDF extraction, import history and bulk export stay gated under ESC-VENDIR-FIELDS (R5). Row cap shown is illustrative.
