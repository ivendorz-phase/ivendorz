**Two governing rules:**

1. **Realize, never re-decide — and never re-specify.** Doc-2/3/4 fixed *what* the system is (FROZEN); Doc-5/6/7 fixed *how* it is realized. Doc-8 realizes *how those are proven correct* — executable suites whose every assertion is an oracle-by-pointer into a frozen declaration. Doc-8 re-decides nothing in Doc-2/3/4 and re-specifies nothing in Doc-5/6/7; a test may never assert behavior the corpus does not declare, nor a stricter/looser rule than the owning contract.
2. **Doc-8 is the conformance harness, but subordinate to its oracle.** Doc-8 *is* the gate that proves Doc-5/6/7 + Code conform, and passes its own Appendix A (`CHK-8-xxx`) before any per-suite freeze — yet holds no authority over what it tests. A red conformance test means the implementation is wrong (fix it) or the corpus is defective (`[ESC-8-CORPUS]`, flag-and-halt) — never that the test should be relaxed. It coins no contract, route, field, permission slug, state, event, audit action, POLICY key, or expected value.

## Ratified decisions (R-set R1–R12)

- **R1 — Program shape: a metastandard + test-discipline-partitioned realizations.** Doc-8 = **Doc-8A** (this metastandard) + realization documents partitioned **by test layer / verification discipline** (Doc-8B…8G), each staged-freeze, gated by Appendix A. The partition **departs from the Doc-5/Doc-6 backend-module letter map by intent** (the Doc-7 precedent): a single conformance concern (the cross-tenant RLS gate, the event-flow suite) spans many modules, so partitioning by discipline keeps each verification concern whole and lets the shared harness freeze once. **The cross-cutting shared document (Doc-8B Test Foundation & Harness) freezes before the suites that consume it.** *(Alternative — per-backend-module partition — rejected: it scatters a single cross-cutting gate across ten documents and duplicates the harness.)*
- **R2 — Realize-never-redecide; the frozen corpus + the realization contracts are the test oracle.** Every assertion traces **verbatim by pointer** to a frozen declaration — the domain *what* (Doc-2 §2/§3/§4/§5/§6/§8/§9; Doc-3 §12; Doc-4J/4L/4M; the 12 invariants + firewall) or a realization contract (Doc-5x field/error/pagination/idempotency; Doc-6x column/constraint/RLS policy; Doc-7x surface behavior). Doc-8 coins **no expected behavior**. Test-infrastructure choices (harness shape, fixture/factory design, assertion style, mock topology) are the only new decisions, and MUST NOT assert behavior absent from the corpus nor a rule stricter/looser than the owning contract. Gaps → `[ESC-8-*]`, escalated, never invented.
- **R3 — Test stack realization = TypeScript test tooling over the fixed Master-Architecture stack; the concrete frameworks are an implementation choice ratified at freeze.** Realize tests against the already-decided runtime stack (Master Architecture / CLAUDE.md §2 — Next.js 15 / TypeScript ≥5 / Prisma / Supabase PostgreSQL / Inngest). The Master Architecture pins **no test framework**, so Doc-8 **coins none** — the specific tooling is an implementation choice the Board ratifies at freeze (or carries via `[ESC-8-TOOLING]`), the Doc-6A R3(b) precedent (one-Prisma-namespace-per-module). The candidate set is carried in the `[ESC-8-TOOLING]` row, not fixed here. No stack runtime element is coined or re-decided.
- **R4 — Conformance is Doc-8's defining function — the inverse of the Doc-6/Doc-7 "consistency-not-conformance" posture.** Where Doc-6/Doc-7 are *consistent-with* (not conformant-to) Doc-5A, **Doc-8 *is* the conformance harness** that gates the Doc-5/6/7 realizations and the eventual Code — but is **subordinate to its oracle**: it cannot change a contract, schema, surface, or corpus declaration. A failing conformance test is dispositioned as **either** a code/realization defect (fix the implementation) **or** a genuine corpus/contract defect (`[ESC-8-CORPUS]` / `[ESC-8-API]` — flag-and-halt → Board, human-approved additive patch at the owning document). **Never resolved by weakening the assertion.** The merge gate (R11) blocks code on any red conformance suite.
- **R5 — Test-data & tenancy realization (Users Act, Organizations Own — Invariant #5; One Module, One Owner — Invariant #7).** Every test seeds and acts through the **same active-org/tenant model** the runtime enforces: server-resolved active org, never a client-trusted org ID. Any tenant-scoped suite seeds **≥2 organizations** (the minimum to prove isolation). Test data is created **through contracts/seed paths, never by hand-mutating another module's tables** — One Module, One Owner holds inside the harness exactly as in production; a suite for module M may not write module N's tables directly. Fixtures realize the standard-column contract (Doc-6 §3) and the role/permission seeds (Doc-2 §7). No fixture invents a column or a role.
- **R6 — The RLS positive / negative / cross-tenant byte-equivalence gate (the load-bearing security gate Doc-6 delegates to Doc-8).** Realize the gate `Doc-6A R8 / §4` hands forward ("the positive/negative/cross-tenant RLS test obligation is **Doc-8's** gate"): **positive** (an authorized actor under the active org sees exactly what the contract grants), **negative** (an unauthorized actor is blocked at the app layer **and** the RLS backstop), **cross-tenant** (org A can never read/list/count org B's tenant-owned rows), and the **non-disclosure byte-equivalence** assertion (Invariant #11 / Golden Rule #5): a blacklisted/excluded vendor's responses, counts, analytics, notifications, logs, and errors are **byte-equivalent** to a non-matched vendor's — `buyer_vendor_statuses` and `link_suggestions` never observable in any vendor-facing surface, view, count, or error. **Mandatory freeze-gate band** (Appendix A); the canonical assertion is defined in Doc-8D (data layer) and re-asserted at the UI in Doc-8G (Doc-7 R8).
- **R7 — Governance-signal firewall & the 12-invariant conformance suites (CLAUDE.md §4/§5; Doc-2).** Each of the **5 firewalled governance signals** and the **12 Core Invariants** has an enforcing suite asserting the firewall holds: Financial Tier never raises Trust Score; Financial Tier does not affect Performance Score; Buyer Approved/Blacklisted never mutates platform-wide scores; no secondary signal dominates trust; no single signal dominates a matching decision; scores auto-calculated under the System actor, never hand-edited; AI suggests but modules decide; capability (4-flag matrix) ≠ label; Financial Tier (capability) ≠ Subscription Plan (commercial); content ≠ presentation; the moat (governed M3 matching/routing + M4 post-award/CRM) is exercised. Defined canonically in Doc-8E; bound by pointer to CLAUDE.md §4/§5 and the owning Doc-2/Doc-4 declarations. No signal, score, or invariant redefined.
- **R8 — State-machine conformance (Doc-4M authority).** Every state machine (RFQ, quotation, verification, subscription, post-award documents, and every other Doc-4M machine) has a suite asserting: every **permitted** transition for the permitted actor in the permitted state succeeds; every **illegal** transition (wrong edge/actor/source state) is rejected; no state label or edge exists in the test that Doc-4M does not declare. Optimistic-UI reconciliation (Doc-7 R9) converges to the server-authoritative state. Defined canonically in Doc-8E. No state or edge coined.
- **R9 — Event & outbox integration conformance (Doc-2 §8 / Doc-4J catalog / Doc-4L flow map; M0-owned outbox).** Suites assert: the **transactional write-plus-emit** atomicity (a business write and its `core.outbox_events` insert commit or roll back together — Doc-6 §7); event payloads conform to the Doc-4J authoritative catalog (zero coined events); cross-module effects propagate **only** via contracts/events and land in the **consuming** module's own schema (no cross-schema write); the outbox → Inngest dispatch lifecycle (`pending → dispatched → archived`) and the Doc-4L fan-out map are honored. No event coined; no cross-module table access introduced by a test.
- **R10 — Frontend & E2E conformance (Doc-7 surfaces).** Suites assert the Doc-7 realizations: component behavior against the shared kit (Doc-7B); the **WCAG-AA + keyboard/focus** baseline (Doc-7 R11); visual-regression of presentation; end-to-end user journeys over the frozen Doc-5 surface (discovery → RFQ → quotation → award → post-award; account/billing; admin console); **per-value-field currency display, default BDT, never assumed** (Doc-2 §0.4); and the **UI non-disclosure byte-equivalence** re-assertion of R6 at the presentation layer (Doc-7 R8 — the Vendor workspace's load-bearing attestation). E2E uses only frozen Doc-5 contracts; it invents no endpoint.
- **R11 — Test isolation, determinism & CI merge-gating.** Every suite is **deterministic** (no wall-clock/random flakiness — a **deterministic ID provider**: the M0 UUIDv7 service (Doc-4B; deterministic per `Doc-6A §7`) fed the **seeded test clock**, or fixed-UUID fixtures where an assertion pins a specific ID), **isolated** (per-test transaction rollback or schema reset; no inter-test state bleed), and **hermetic** (no live external network — every out-of-wire boundary mocked: Supabase Storage, Realtime, Resend email, PostHog, the Inngest dispatch surface, the M9 AI providers). The **CI gate blocks merge on any red conformance suite** (R4); flaky tolerance is zero — a non-deterministic test is a defect. AI-generated code clears the Doc-8 gate **plus** AI Coding Supervisor or human review before merge (CLAUDE.md §8); the gate never substitutes for the review.
- **R12 — Out-of-test boundary (the verification analog of the out-of-wire / out-of-DB / out-of-frontend fence).** Tests own **no production state**: the test database is ephemeral; test doubles, fixtures, factories, and mocks are disposable and never a source of production truth; a test never writes to production data or a live external service. Mocked boundaries (Storage / Realtime / Resend / PostHog / Inngest / AI providers) are **simulated, never live**. A test may **observe** but never **author** the corpus contract it verifies. **Flag-and-halt if any test is proposed as a source of production truth, as the authoritative definition of a behavior, or as a backdoor that asserts behavior the corpus does not declare.**
