## Decisions proposed for ratification at structure freeze

- **R1 — Out-of-wire boundary** (Doc-5B/5C/5D/5E/5F/5G R1 precedent; authority `Doc-5A §1.3/§5/§11`). Doc-5J realizes only the caller-facing HTTP surface (platform-staff Admin). **2 contracts have no caller wire** (§10): the System workers `admin.expire_ban.v1` (21.5 — `ban_actions` expiry timer) and `admin.process_import_job.v1` (21.5 — async import job). In addition, three classes of internal leg are fenced out-of-wire: (a) the **System auto-queue legs** of the dual-template `admin.create_moderation_case.v1` and `admin.queue_verification_task.v1` (each is 21.6 Admin / 21.5 System — only the Admin caller leg is realized; `Doc-5G create_fraud_signal` dual-leg precedent); (b) the **cross-module write-via-service legs** (R5 — verification decision → Trust, ban reflection → Marketplace, link columns → Operations, category/import seed → Marketplace); (c) the **`VendorBanned` outbox delivery** (Admin authors production/delivery — Doc-4A §4.4; M0 outbox). **Flag-and-halt if any wire is proposed for them.**
- **R2 — Platform-staff Admin actor only; no tenant User, no public** (the M8 differentiator, and the inverse of M4's User-only surface). M8 has **one actor class on the wire — the platform-staff Admin** (Doc-4J H.3 — the platform-staff slug space, `Doc-4A §5.6`). There is **no active organization context** — `Iv-Active-Organization` is **not carried** (platform-staff actions are not org-scoped); **Stage-5 DELEGATION is n/a throughout** (no representative-org scenario — Doc-4J H.3). **No tenant-User surface. No public/anonymous surface.** System out-of-wire (R1). Actor type is server-determined; no field/header asserts it (`Doc-4A §9.7`).
- **R3 — `admin` route prefix; `admin.` Contract-ID token (no split)** (`Doc-5A Appendix B.1` registers the M8 route namespace `admin`, `Doc-2 §0.3`; the Doc-4J Contract-ID operation token is `admin.<operation>.v1` with error-code namespace `admin_`, `Doc-4J §B`/`Doc-4A Appendix B.2`). **The route prefix and the token stem are the same string `admin`** — unlike M4's deliberate `operations`/`ops.` split, M8 has no split. Path grammar (§5.3) derives from the route prefix `admin`. Doc-5J coins neither.
- **R4 — No token invented.** Endpoints bind existing Doc-2 §7 platform-staff slugs (`staff_can_moderate_rfq`, `staff_can_ban`, `staff_can_manage_categories` [category-suggestion decisions ONLY], `staff_can_verify`; `staff_super_admin` flagged override — Doc-4J H.3), §9 audit actions, and the §8 event catalog; carried gaps bound by pointer and **escalated, never invented**: `[ESC-ADM-SLUG]` (Doc-2 §7), `[ESC-ADM-AUDIT]` (Doc-2 §9), `[ESC-ADM-POLICY]` (Doc-3 §12.2), `[ESC-ADM-EVENT]` (Doc-2 §8) — `CHK-5A-121/154`; `Doc-4A §6.4/§18.2`.
- **R5 — Admin decides; the owning module owns (the M8 signature firewall; Invariant #7 + Red Flag #8 — Admin must never bypass an owning module's domain).** M8 owns its own aggregates — `moderation_cases`, `ban_actions`, `category_suggestions`/`missing_vendor_suggestions`/`link_suggestions`, `import_jobs`, `verification_tasks`, `outreach_campaigns` — and writes only those. Every **cross-module effect** of an Admin decision is realized as an **in-process service call to the owning module** (the out-of-wire write-via-service leg, §10/R1): the verification decision is recorded **via Trust's service** (`verification_tasks ≠ trust.verification_records` — DR-ADM-TRUST), the ban reflection **via Marketplace** (`VendorBanned` consumer effect — DR-ADM-MKT/DD-3), the confirmed link columns **via Operations** (DR-ADM-OPS/A-03), the category/import-seed writes **via Marketplace** (DR-ADM-MKT). **No cross-module table write, no cross-module foreign key, no Admin HTTP surface for another module's data** (`CLAUDE.md §3`; Doc-4A §5.3/§4.3).
- **R6 — Non-disclosure firewall (Invariant #11).** `link_suggestions` content (and all platform-staff-only records) is **never tenant-visible** through any surface; every M8 read is **Staff-Internal**; a protected-fact-gated or unauthorized read collapses to a uniform **`NOT_FOUND`** (no side-channel — `Doc-4A §7.5`; `Doc-4J H.4` §7.5). The vendor-facing side of a link (where one exists) is owned and realized by the owning module (Operations/Marketplace), never here.
- **R7 — Procurement moat; Admin governs, Admin does not procure (DF/moat).** M8 makes **no matching / routing / ranking / supplier-selection / award / eligibility decision** (RFQ — Doc-4E; DR-ADM-RFQ references the RFQ subject by UUID only). **Vendor outreach (BC-ADM-6) is informational acquisition only** (Doc-4J H.7/§F-ADM-6) — no procurement surface. **Flag-and-halt if a matching/routing/award surface is proposed.**
- **R8 — Governance-signal / Trust firewall (Invariant #6).** M8 **computes, owns, and stores no trust / performance / verification / financial-tier score** (Trust — DR-ADM-TRUST). The Admin **verification workflow task is not the Trust verification record** (`verification_tasks ≠ trust.verification_records` / `≠ trust.verification_decisions`); Admin owns the workflow, Trust owns the decision and any score. No governance signal is written on any M8 wire.
- **R9 — Authoritative event catalog; single emission (Doc-4J; CLAUDE.md §9).** M8 is the **authoritative event catalog owner**, yet emits exactly **one** Doc-2 §8 domain event today: **`VendorBanned`** (BC-ADM-2 `issue_ban`; sole Admin producer — Doc-4J H.7). **BC-ADM-1/3/4/5/6 emit No Event** — every other M8 mutation is a **state + §9 audit** action. The event is emitted to the **M0 transactional outbox** by the realizing command (business write + event insert, one transaction — `Doc-4B`); consumers (Marketplace reflect, Communication notify, Trust Protection freeze) own their effects. `Doc-5A §11` carries **no** caller-facing webhook/push surface; any future Admin event carries `[ESC-ADM-EVENT]`, never coined here.
- **R10 — Job submit/accept shape `[realization convention — resolve from frozen Doc-4J at content]`.** `admin.submit_import_job.v1` (and `admin.run_outreach_campaign.v1`) enqueue work the System processes out-of-wire (`process_import_job`, §10). Whether the enqueue realizes as **`201` created-queued** (returning the `queued` resource) or **`202` accepted-then-processing** (Doc-5A §10) is **resolved at content from the frozen Doc-4J §F response shape** — the single genuine async-shaped question. The structure does not decide it; the content pass binds it to the frozen source and marks any transport-silent choice `[realization convention]`.
